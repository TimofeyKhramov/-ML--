**Описание ML-сервиса**

ML-сервис позволяет создавать файлы о финансовой отчётности компании посредством анализа изображений бумажных чеков

**Шаг №1. Анализ требований**

1. Нужно разработать Web-приложение: личный кабинет пользователя ML-сервиса
2. Бизнес сущности и действия пользователя:
   1. Возможность регистрации для использования сервера и дальнейшей авторизации: пользователь должен иметь логин и пароль
   2. Возможность просмотра баланса: графический интерфейс должен иметь окно с отображением баланса
   3. Возможность пополнения баланса и списания средств: приложение должно позволять проводить транзакции
   4. Возможность выполнить запрос к ML-сервису: должна быть ML-модель и ML-задача, позволяющие проводить необходимые действия и генерировать ответ
   5. Возможность просмотра истории загруженных данных и предсказаний по ним с указанием суммы списанных средств
   6. Возможность выполнять запросы через REST интерфейс
      
**Шаг №2. Выделение сущностей**

Ниже приведено описание основных классов и их ответственности (функционала)

Класс №1. Пользователь (User)

Осуществляет взаимодействие с ML-сервисом посредством выполнения запросов через REST интерфейс.
Обладает идентификатором, данными для авторизации (логин, пароль), возможностью просматривать баланс и возможностью скачивания итогового файла

Класс №2. ML-модель (MLModel)

Модель позволяет создавать финансовую отчётность на основе кассовых чеков. На вход модель принимает фото кассового чека. Поддерживаемые форматы: jpg, jpeg, png. Далее осуществляется перевод изображения в текст (image-to-text). На выходе модель возвращает json-файл  со следующими ключами: 
  1. Дата
  2. Наименование расхода
  3. Сумма
  4. Поставщик
  5. ИНН
     
Далее планируется перевод в csv-файл и отправка конечного ответа пользователю
Возможные модели: FRED-T5-large-ods-ner-2023 (вес около 3.3Гб), 
LightOnOCR-1B-1025 (вес около 3Гб)

Класс №3. ML-задача (ML-task)

Данный класс предназначен для определения статуса (описание в коде) и типа задачи (на данный момент есть один тип)

Класс №4. Транзакция (Transaction)

Данный класс позволяет пользователю пополнять баланс и списывать средства для выполнения задачи.
Классы DebitTransaction и AddTransaction наследуют от данного класса.
